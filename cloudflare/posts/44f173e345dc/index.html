Claude Code

命令

基本用法

  • 在会话里直接敲/弹出所有可用命令，支持字母过滤。

  • 命令只有放在消息开头才会被识别，后续文字作为参数（如/plan 修复登录的bug）。

  • 参数约定：<arg>表示必填，[arg]表示可选。

  • 部分命令可见性取决于平台、套餐和环境（如/desktop需macOS/Windows + 订阅账号，/upgrade需Pro/Max套餐）。

  • 特殊命令类型

    • Skill（技能）：内置提示词，Claude可自动调用。
    • Workflow（工作流）：动态工作流，拆分任务给子代理后台并行执行。

账号与用量

命令 作用
/login / /logout 登录 / 退出Anthropic账号
/usage 查看会话成本、套餐用量上限和活动统计（别名/cost, /stats）
/usage-credits 配置用量额度，撞上限也能继续用（原/extra-usage）
/upgrade 打开升级页切换更高套餐
/privacy-settings 查看和更新隐私设置（仅Pro和Max可用）
/passes 送朋友一周免费Claude Code（符合条件才可见）

💡 提示：重度用户常看/usage，可按技能、子代理、插件等细分查看Token消耗。

命令 作用
/init 生成CLAUDE.md说明文档，让Claude通读项目结构

💡 提示：这是新项目第一条该敲的命令，为后续对话提供上下文。

开发常用

命令 作用
/plan [description] 进入计划模式，可直接带任务
/model [model] 切换模型并存为默认，支持左右键调推理强度
/effort [level] 设置推理强度（low/medium/high/xhigh/max/ultracode）
/context [all] 可视化当前上下文占用并给优化建议
/memory 编辑CLAUDE.md，开关自动记忆，查看条目
/permissions 管理工具权限规则（别名/allowed-tools）
/compact [instructions] 压缩当前对话腾出上下文，可指定侧重点
/btw <question> 问一个不写入正式历史的问题
/goal [condition] 设定目标，Claude跨多轮持续执行直到达成
/advisor [model] 开关顾问工具，关键节点叫第二个模型出主意
/fast [on 、off] 开关快速模式
/mcp 管理MCP服务器连接和OAuth授权
/agents 管理子代理（Subagent）配置
/hooks 查看工具事件的钩子配置
/rewind 回滚对话和/或代码到检查点（别名/undo）

💡 核心技巧：

  1. /plan适合大改动前理清思路；/model + /effort搭配使用，简单任务低强度省钱，复杂任务用max或ultracode。
  2. 上下文快满时用/context查看占用，用/compact压缩；插话用/btw避免污染主线。
  3. /permissions管好权限，配合/fewer-permission-prompts减少确认弹窗。
  4. /goal适合“跑通测试为止”这类持续性任务。

会话管理

命令 作用
/clear [name] 开新对话，旧对话保留在/resume（别名/reset,/new）
/resume [session] 恢复对话或打开选择器（别名/continue）
/rename [name] 给当前会话改名
/recap 生成当前会话一句话摘要
/export [filename] 导出当前对话为纯文本
/copy [N] 复制最近N条回复
/stop 停掉当前后台会话
/exit 退出CLI或脱离后台会话（别名/quit）
/insights 生成使用情况分析报告
/add-dir <path> 临时添加可访问的工作目录
/cd <path> 把整个会话挪到新目录并保留缓存

💡 注意：/clear是另起炉灶，/compact是同对话腾地方；/add-dir是额外授权目录，/cd是整体搬迁。

并行任务

命令 作用
/background [prompt] 切到后台跑，释放终端（别名/bg）
/batch <instruction> Skill拆分大型改动为独立单元，多子代理并行提PR
/fork <directive> 派后台子代理继承完整对话去干活
/branch [name] 拉分支换方向试，原对话可找回
/tasks 查看管理后台任务（别名/bashes）
/loop [interval] [prompt] Skill按周期重复跑提示词
/workflows 管理工作流进度面板

💡 核心技巧：

  1. /batch是大杀器，适合大规模迁移重构，自动拆单、独立worktree、互不打架。
  2. /fork是派人后台干，/branch是自己换条路。
  3. /loop适合定时检查CI等监控任务。

交付检查

命令 作用
/diff 交互式查看未提交改动
/code-review [...] [--fix] Skill审查正确性bug，--fix落地修改，ultra走云端深度审查
/review [PR] 审查GitHub PR
/security-review 安全审查（注入、鉴权、泄露等）
/simplify [target] Skill只做清理简化，不找 bug
/verify Skill真正运行app验证改动效果
/run Skill启动并驱动app看效果
/run-skill-generator Skill教/run和/verify如何构建启动项目
/autofix-pr [prompt] 云端盯着PR，CI挂或有评论自动推修复

💡 建议流程：提交前/diff → /code-review → 确认无误--fix。首次用/verify前先跑/run-skill-generator。涉及外部输入的改动多跑/security-review。

worktree

# 在仓库根目录下，为feature-auth功能创建一个独立的worktree并启动Claude
claude --worktree feature-auth

# 再开一个终端，为bugfix任务启动另一个独立会话
claude --worktree bugfix-123
  • 默认位置：worktree默认创建在仓库根目录下的.claude/worktrees/<你给的名字>/。
  • 默认分支：新worktree默认基于仓库的远程默认分支（如origin/main）创建。你也可以在设置中配置worktree.baseRef: "head"来让它基于当前的本地HEAD。

安装插件

# 添加官方Skill市场
/plugin marketplace add anthropics/

/plugin install document-skills@anthropic-agent-skills

npx skills add https://github.com/anthropics/skills --skill pptx
